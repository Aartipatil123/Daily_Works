What I've Learned About HTML and CSS

Lesson 1

World Wide Web

The web is basically made up of a bunch of computers that communicate with each other. Majority of the content on the web is simply a collection of HTML (Hypertext Markup Language) documents.

How it Works?

Your browser makes HTTP (Hypertext Transfer Protocol) requests via the Internet to servers (servers are computers that are designed to live in drawers and host files), that respond to the requests with files that your browser displays.

Elements, Tags and Attributes

HTML elements tell a web browser how to display content in a web page. HTML elements usually have opening and closing tags, with content between them, that tell the browser the type of each element. Attributes give additional information to the browser and always have a value. For example, the ancor <a> tag, which is used to add links, takes an attribute href, that defines a URL to link to. The image tag <img> , which is used to include images, takes the attribute src , that defines the source/URL of the image.

Whitespace

In HTML all whitespace - new lines, tabs, spaces - turn into a single space. In order to create line break, you need to use, for example, a break tag <br> or a paragraph tag <p>. NOTE: Since the <p> tag is not a void tag like the <br> tag, it needs to have a closing tab as well.

Span and Div

<Span> and <div> elements both simply contain content, but you can add styles them by using CSS. The important difference between the two is that <span> elements are inline and <div> elements are block.

Lesson 2

HTML

HTML controls the structure of a web page. Each HTML tag creates an element that the browser uses to display the page. An element is created from the starting tag to the end tag, and everything in between is content. The content can be text, another element or even empty. Each tag has a name, that defines what kind of element will be created. It could be a paragraph, a document division or an image. Each tag can also have attributes with values.

CSS

CSS controls the style and how the elements look on a page. For example, the size of the font, colour, position, background, boarders etc.

DOM

DOM (Document Object Model) refers to the tree-like structure of a page. In a browser, the tree-like structure shows up as a series of nested boxes.

Boxes Everywhere

Everything in a webpage is actually a box. (There is also grid in the background that helps make sure everything is properly aligned).
Programmer advice on boxes: You can start out with a pencil and simply draw out the boxes you need, starting from the big and working down to the small ones.

Lesson 3

CSS Basics

Selector is the element that you want to apply certain CSS rules to. For example, if selector is <h1>, it tells the browser to apply the rules to every <h1> element. Everything between the curly braces will be interpreted as a rule that is applied to every <h1> element, if we want to apply style to certain <h1> elements, we can use class selectors.
The line of code that says background-color:black; is a declaration. Background-color is a property and black is the value of that property.

Three Ways to Include CSS Styling

1. Write CSS in the head of your HTML (small projects). 2. Link your HTML to a separate CSS file (larger projects - much more organised). 3. Write your style inline with your HTML (bad idea).

The Box Model

All HTML elements are boxes and each box has 4 elements: content, padding, boarder and margin. The padding clears an area around the content and it’s affected by the background colour of the box. Boarder goes around the padding and the content. The boarder is inherited from the colour property of the box. Margin (like space between boxes). Margin clears an area around the border. The margin doesn’t have a background colour and is completely transparent.

Box Sizing and Positioning

It can be hard to get the size of a box just right due to many components. There are two techniques that help with sizing issues:
1) Set sizes in terms of percentages instead on pixels.
2) Set the box-sizing attribute to boarder-box for every element. <Div> elements are block, so by default they take up the entire width of a page. Adding the rule display:flex; lets <div> elements appear next to each other.

Code, Test, Refine

1) Look for natural boxes and places that are distinct.
2) Look for repeated styles and semantic elements (work on the tags).
3) Think about how you can relate this to HTML elements and write your HTML. Consider the size of boxes and how to position them on the screen.
4) Apply styles, from biggest site-wide elements to smallest.
5) Fix things, open in different browsers, resize screen size etc.
